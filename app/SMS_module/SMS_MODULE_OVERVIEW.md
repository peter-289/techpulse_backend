# Software Management Module (SMS)

## Architecture
- `domain/`: pure entities, value objects, domain services, events, exceptions.
- `application/`: command/query models, use-case services, and abstract ports.
- `infrastructure/`: SQLAlchemy repository + mappers, S3 adapter, queue adapters.
- `api/`: FastAPI router, request/response schemas, and entitlement gatekeeper dependency.

## Key Guarantees
- Artifact lifecycle uses `ArtifactStatus` (`UPLOADING`, `ACTIVE`, `DELETED`, `QUARANTINED`).
- Artifact is moved to `ACTIVE` only via `process_malware_scan_success(...)`.
- Download links are presigned URLs generated by `StoragePort` adapters.
- Access is guarded via `SubscriptionPort.verify_access(user_id, software_id)`.
- Version optimistic locking is configured with SQLAlchemy `version_id_col` (`lock_version`).

## Host Integration Contract
- Implement `SubscriptionChecker` (`application/ports.py`).
- Provide factories in `create_sms_app(...)` for:
  - `SoftwareCommandService`
  - `DownloadService`
  - `SubscriptionPort`

## Notes
- Queueing supports Celery or Arq through `MalwareScanQueuePort` adapters.
- Domain and application layers do not import Auth/Billing/Subscription modules directly.
